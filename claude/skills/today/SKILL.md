# today — 1시간 타임박스 브리핑

`~/.local/share/ct/todos.md`를 읽어서 우선순위 + 마감일 + 가용 시간 기반으로 오늘의 할 일을 보여준다.

## 동작

1. `~/.local/share/ct/todos.md` 읽기
2. 모든 섹션(Calendar, Sprint, GitHub, Manual)의 미완료 항목(`- [ ]`) 수집
3. 정렬 및 타임박싱
4. 브리핑 출력

## 정렬 기준 (우선순위 내림차순)

1. **마감일이 오늘 이전** (overdue) — 무조건 최상위
2. **p:high + 마감일 가까운 순**
3. **p:medium + 마감일 가까운 순**
4. **마감일 없음 / p:low**
5. **PR 리뷰** — 팀 블로킹이므로 높은 우선순위로 취급

## 타임박싱

- 기본 예산: **1시간** (60분)
- Calendar 섹션의 회의 일정은 참고 정보로 표시 (타임박스에서 차감하지 않음)
- 각 항목에 예상 소요 시간을 배분:
  - PR 리뷰: 15분
  - Sprint 태스크 (규모 1-2): 20분, (규모 3): 30분, (규모 5+): 40분
  - GitHub 이슈: 15분
  - Manual todo: 10분 (기본값)

## 출력 형식

```
# 2026-04-03 (목) — 1h Time Box

## 오늘 일정
- 10:00-10:30 스탠드업
- 14:00-15:00 스프린트 리뷰

## 할 일 (60분)

1. [15m] PR: 리뷰 제목 — repo
   → https://github.com/...

2. [20m] TASK-123: 태스크 제목 (p:높음, due:04/07)
   → https://notion.so/...

3. [10m] tracking plan 확인 (due:04/07)

4. [15m] 마일스톤 설정

---
합계: 60m / 60m
남은 항목: N개 (내일 이후)
```

## 데이터가 없을 때

todos.md가 비어있거나 동기화되지 않았으면:
> "데이터가 없습니다. `/today-sync`로 먼저 동기화하세요."

## 읽기 전용

이 스킬은 todos.md를 **수정하지 않는다**. 보여주기만 한다.
