# today — 근무시간 타임박스 브리핑

`~/.local/share/ct/todos.md`를 읽어서 우선순위 + 마감일 + 가용 시간 기반으로 오늘의 할 일을 보여준다.

## 동작

1. `date '+%H:%M'`로 현재 KST 시간을 확인
2. `~/.local/share/ct/todos.md` 읽기
3. 모든 섹션(Calendar, Sprint, GitHub, Manual)의 미완료 항목(`- [ ]`) 수집
4. Calendar 회의 시간을 가용 시간에서 차감
5. **현재 시간 기준** 정렬 및 타임박싱
6. 브리핑 출력

## 정렬 기준 (우선순위 내림차순)

1. **마감일이 오늘 이전** (overdue) — 무조건 최상위
2. **p:high + 마감일 가까운 순**
3. **p:medium + 마감일 가까운 순**
4. **마감일 없음 / p:low**
5. **PR 리뷰** — 팀 블로킹이므로 높은 우선순위로 취급

## 타임박싱

- 근무시간: **KST 08:00 – 17:00** (9시간)
- 마지막 1시간(16:00–17:00)은 **정리/버퍼** 시간으로 예약
- 기본 예산: **8시간** (480분) − 회의 시간
- Calendar 섹션의 회의 시간을 가용 시간에서 **차감**하고, 일정 사이의 빈 시간에 작업을 배치
- 각 항목에 예상 소요 시간을 배분:
  - PR 리뷰: 15분
  - Sprint 태스크 (규모 1-2): 20분, (규모 3): 30분, (규모 5+): 40분
  - GitHub 이슈: 15분
  - Manual todo: 10분 (기본값)

### 현재 시간 반영 규칙

- 타임박싱 시작 시점은 **현재 시간과 08:00 중 더 늦은 시간**
- 현재 시간 이전의 회의는 "지나간 일정"으로 분류하고, 가용 시간 차감에서 **제외**
- 현재 시간 이후의 회의만 가용 시간에서 차감
- 헤더에 실제 타임박싱 범위를 표시: `(HH:MM–17:00)`
- 근무시간 종료(17:00) 이후에 실행하면: `"오늘 근무시간이 종료되었습니다. 내일 다시 확인하세요."`

## 출력 형식

```
# 2026-04-03 (목) — Work Day Plan (14:20–17:00)

## 지나간 일정
- ~~10:00-10:30 📅 스탠드업~~ (완료)
- ~~13:00-14:00 📅 스프린트 리뷰~~ (완료)

## 타임라인

| 시간 | 항목 |
|------|------|
| 14:20-14:40 | TASK-123: 태스크 제목 (p:높음) |
| 14:40-14:55 | PR 리뷰: 리뷰 제목 — repo |
| 14:55-15:10 | PR 리뷰: 리뷰 제목2 — repo |
| 15:10-15:30 | TASK-456: 태스크 제목 |
| 15:30-16:00 | PR 리뷰: 리뷰 제목3 — repo |
| 16:00-17:00 | 🔄 정리/버퍼 |

## 요약
- 지나간 회의: 1h30m
- 남은 회의: 0m
- 작업 가능: 1h40m (14:20–16:00)
- 배정됨: 1h40m / 1h40m
- 남은 항목: N개 (내일 이후)
```

## 데이터가 없을 때

todos.md가 비어있거나 동기화되지 않았으면:
> "데이터가 없습니다. `/today-sync`로 먼저 동기화하세요."

## 읽기 전용

이 스킬은 todos.md를 **수정하지 않는다**. 보여주기만 한다.
